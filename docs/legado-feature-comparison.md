# MoRealm 与 Legado 功能完整对照

> 对照基准：`D:\temp_build\legado` 与 `D:\temp_build\sigma\legado` 当前源码、README、主要 UI / model / service / data 目录，以及 MoRealm 当前工作区源码。本文只记录已经在 MoRealm 源码中存在的能力；规划项不会写成已完成。

## 结论总览

MoRealm 已经覆盖了 Legado 的核心阅读链路的一部分：本地书架、TXT/EPUB/PDF/漫画压缩包阅读、书源导入与搜索、正文解析、替换净化、书签、阅读设置、TTS、WebDAV/备份、主题与日志。并且在现代 UI、Compose 渲染、年度报告、主题编辑、卡片化体验、部分漫画压缩包随机访问等方面已经有明显差异化。

但如果以 Legado 的“全功能阅读平台”作为标杆，MoRealm 仍未完全追平。Legado 的优势集中在：书源生态深度、订阅/RSS、发现页、源调试器、远程 Web 服务、二维码/URL 协议导入、字典规则、TXT 目录规则管理、换源/换章节源、书籍导出、键盘辅助、文件关联、更多成熟配置与边界处理。

## 详细对照

### 1. 阅读引擎与章节切换

Legado 的阅读核心围绕 `TextPageFactory` 和多个 `PageDelegate`，对当前章、上一章、下一章有成熟的窗口管理。尤其上下滚动模式不是简单拼接字符串，而是维护章节边界和页面委托，因此短章节、封面、目录页等边界场景更稳。

MoRealm 当前已经具备：

- Canvas 阅读引擎，支持分页、仿真/滑动等动画、TTS 高亮、选中文本、背景图。
- WebView 阅读引擎，可承载完整 HTML/CSS 排版。
- 上下滚动渲染器，支持追加下一章，并已针对“短章节/封面/目录不自然衔接”做过修正。
- `RenderedReaderChapter` 原子渲染状态，降低切章时进度闪烁和白屏。

评价：MoRealm 架构更现代，但 Legado 的章节窗口模型仍值得继续学习。下一步建议把滚动模式从“拼接内容”升级为“多章节段落列表 + 可见章节锚点”，这样 cover、目录、超短章、图片章都能自然衔接。

### 2. 本地格式与大文件

Legado 本地白名单包含 `txt|epub|umd|pdf|mobi|azw3|azw`，压缩包包含 `zip|rar|7z`。它通过 libarchive 处理压缩包，并有本地导入模型。

MoRealm 当前状态：

- TXT：块读取、超大章节拆分、支持自定义章节正则。
- EPUB：epublib 随机访问解析，封面/元数据提取。
- PDF：Android `PdfRenderer` 渲染。
- 漫画压缩包：`.zip/.cbz` 走 `AndroidZipFile` 随机访问，`.rar/.cbr/.7z` 走 libarchive。
- MOBI/AZW3/UMD：已有解析器入口和格式识别。

评价：格式面已经接近 Legado；漫画大压缩包方向甚至更贴近当前用户需求。仍需补：导入 `.zip` 时判断“图片压缩包”还是“书籍压缩包”，避免把包含 EPUB/TXT 的 zip 当漫画空解析。

### 3. 书架与导入

Legado 有成熟的本地扫描、远程导入、书架样式、分组和批量管理。MoRealm 当前书架具备现代化卡片、文件夹封面拼图、批量导入、置顶、分组、两阶段元数据补全。

近期 MoRealm 已修复一个关键差异：导入大目录时，如果父目录只是容器，不再把父目录本身也导入成一本“巨型书”，而是递归导入其内部真实书籍/书籍文件夹。这更符合 Legado 智能扫描的思路。

剩余差距：

- 缺少 Legado 那种远程书籍服务器导入体验。
- 缺少更细的导入预览、过滤、冲突合并 UI。
- 漫画/小说混合目录的分类策略还可以更智能。

当前新增能力：MoRealm 分组支持 `autoKeywords`。用户可在新建/编辑分组时填写关键词，系统会用书名、作者、简介、分类/标签、字数标签、路径、来源名进行包含匹配；导入本地书、搜索结果加入书架、详情元数据更新时会自动把未分组书籍放入匹配分组。也可以在分组管理里手动触发“按关键词重新归类”。

### 4. 书源生态

Legado 的最大壁垒是书源生态：书源、发现、订阅、源登录、验证码、调试器、规则说明、Web 端源编辑/调试、换源、章节换源等。

MoRealm 当前已经完成主链路：

- Legado 书源 JSON 导入。
- 多源搜索。
- 书籍详情、目录、正文解析。
- CSS/XPath/JSONPath/Regex/Rhino JS 能力。
- Cookie、登录字段、HTTP 配置字段有数据库支撑。
- 多书源导入已改为“解析后逐条入库并显示进度”。

差距：

- 发现页规则未完整落地。
- 源调试器缺失，排查书源问题不如 Legado。
- 登录/验证码/WebView 登录链路不完整。
- 换源/章节源体验不完整。
- 订阅源/RSS 缺失。

建议：书源部分不应一次性追全。优先级应是“源调试器 > 登录/Cookie > 发现页 > 换源 > RSS”。

### 5. 搜索与结果展示

MoRealm 的搜索 UI 更现代，之前已经针对“书源特别多时结果看不到”讨论过保留书源效果的折叠方案，也已改进多书源导入的增量反馈。

对照 Legado，还需要：

- 搜索结果按书源/书籍聚合折叠。
- 慢源、失败源、空结果源状态可视化。
- 一键禁用异常源。
- 搜索结果进入详情后，URL 缺 scheme、JS 语法异常等错误要做兼容修正和用户提示。

Sigma 对搜索做了额外增强：`SearchModel` 和 `BookList` 会把 `kind` 纳入过滤/命中逻辑，搜索结果展示也会显示 kind 标签。MoRealm 目前已有 `kind/customTag/category/wordCount` 字段，并在自动分组匹配中使用这些字段；但搜索结果 UI 的标签筛选、kind 命中排序还没有完整追平 Sigma。

### 5.1 缓存章节与目录中间插章

Legado 相关问题：如果离线缓存以目录序号或刷新后的章节位置为主，一旦目录在中间插入新章，插入点之后的缓存可能全部错位或失效，导致重复下载。

MoRealm 当前判断：主缓存 key 是 `chapter_content_{sourceUrl}_{chapterUrl}`，也就是按章节 URL 缓存，而不是按章节序号缓存。因此只要书源的章节 URL 稳定，中间插章不会导致插入点之后的缓存整体失效。涉及位置：`CacheBook.kt` 和 `CacheBookService.kt`。

已补强点：后台刷新目录之前只比较章节数量，无法发现“数量相同但中间替换/插入后又删除”的变化。现在目录刷新会比较每章 title/url/isVolume，只要顺序或 URL 发生变化就更新本地目录。注意：如果某个书源的章节 URL 本身就是按序号生成且会随插章变化，那么任何按 URL 缓存的方案也会受影响，需要额外做“标题 + 正文指纹”迁移，这一层 MoRealm 目前还没有实现。

### 6. 阅读设置与主题

Legado 的阅读配置非常细，包括字体、背景、行距、段距、加粗、简繁、点击区域、翻页、提示栏等。MoRealm 当前在 Compose 设置体验上已经较好，并具备：

- 字号、行距、段距、页边距、顶部/底部边距。
- 自定义 CSS、快捷 CSS 预设。
- 自定义字体、标题字体、字重。
- 翻页动画、上下滚动、双页。
- 状态栏/章节名/时间电量显示配置。
- 主题编辑、主题导入导出。
- “纸上”主题已回到橘色暖纸风格。

评价：常用设置已接近甚至更顺滑；Legado 的点击区域、提示栏、键位辅助等细分设置仍更丰富。

### 7. TTS 与听书

Legado 支持系统 TTS、HTTP TTS、朗读服务、音频播放。MoRealm 已具备：

- 系统 TTS。
- Edge TTS 中文神经语音。
- HTTP TTS API。
- 通知栏控制、音频焦点、睡眠定时。
- 朗读跟随滚动、段落控制、失败回退。

评价：纯朗读体验 MoRealm 已经很强，Edge TTS 是超越点。若要追 Legado 的音频播放，需要补“有声书/音频源/播放列表/断点”体系。

### 8. 替换净化、TXT 目录、字典

Legado 有三类规则体系：替换规则、TXT 目录规则、字典规则。MoRealm 当前：

- 替换规则：有实体、仓库和管理页面，阅读中可应用。
- TXT 目录规则：有实体/仓库/自定义正则能力，但独立管理体验不完整。
- 字典规则：缺失。

建议：优先完善替换规则的分组、导入导出、作用范围；TXT 目录规则补管理 UI；字典规则可以排在 RSS/源调试器之后。

### 9. WebDAV、备份与 Web 服务

Legado 拥有 WebService、NanoHTTPD、WebSocket、Web 书架、Web 源调试能力。MoRealm 当前拥有：

- WebDAV 配置、测试。
- 本地 ZIP 备份/恢复。
- 自动备份开关。
- NanoHTTPD/WebSocket 依赖已引入，但完整 Web 服务功能尚未系统呈现。

评价：备份可用；Web 端能力是明显差距。如果要追 Legado，可以先做“小而美”的 WiFi 传书，再扩展 Web 书架和源调试。

### 10. 产品化与差异化

MoRealm 已有几个 Legado 没有或不突出的方向：

- Compose + Material 3 的现代 UI。
- 年度阅读报告，动态轮播并保存 PNG。
- 主题编辑器和内置主题动效更现代。
- 工程化日志体系更强，含 ANR/崩溃/文件日志。
- 阅读器内引用卡片分享能力。
- 更积极支持 PDF/漫画压缩包统一书架体验。

这些是 MoRealm 不必照搬 Legado、可以形成自己产品气质的地方。

## 与 Legado 相比的主要缺口

### P0：影响核心看书体验

1. 上下滚动 EPUB/PDF 边界继续 Legado 化：章节窗口、多章节锚点、短章自动续接、PDF 底部判定。
2. 书源错误兼容：URL 补 scheme、JS eval 包装、搜索结果详情防崩、错误可读化。
3. 源调试器基础版：输入书源和关键词，显示请求、响应、规则命中、JS 错误。
4. 换源/章节换源：搜索结果和书籍详情需要可切换来源。

### P1：追平 Legado 高频功能

1. 发现页规则。
2. 书源登录/WebView 登录/验证码。
3. TXT 目录规则管理页面。
4. 替换规则导入导出、分组和作用范围。
5. WiFi 传书和基础 Web 书架。
6. 全局书签/阅读记录页。

### P2：扩展生态功能

1. RSS/订阅源。
2. 字典规则。
3. 二维码扫描和完整 `legado://import/{path}?src={url}` 协议兼容。
4. 书籍导出服务。
5. 键盘辅助、蓝牙/遥控器按键细分。
6. 漫画专用滤镜、Webtoon 专用设置、电子纸模式。

## 建议路线图

### 第一阶段：把当前痛点打磨到稳定

- 阅读器章节切换：采用 Legado 的“当前/上一/下一章节窗口”思想，不再只依赖内容拼接。
- PDF/漫画滚动：严格区分“预加载下一页/章”和“确认到达底部”，避免未到底就跳书。
- 书源详情：修复无 scheme URL、JS 字符串 eval、特殊标题搜索结果打开失败。
- 搜索结果：按书源折叠展示，保留书源视觉效果，同时避免结果被大量源状态淹没。

### 第二阶段：补齐 Legado 书源生态的关键环

- 做源调试器。
- 做 WebView 登录和 Cookie 持久化。
- 做发现页。
- 做换源/章节换源。
- 做规则导入类型统一入口。

### 第三阶段：建立 MoRealm 自己的优势

- 年度报告继续增强：多页故事卡、分享图、阅读人格、月份热力图。
- 漫画阅读增强：Webtoon、双页、滤镜、电子纸、长图分割。
- 本地导入智能分类：zip 内含图片则漫画，内含 epub/txt/pdf 则展开导入书籍。
- Web 端能力：WiFi 传书、Web 书架、Web 源调试。

## 最终判断

MoRealm 目前不是“完整超越 Legado”，但已经不是简单仿品。它在现代 UI、阅读卡片体验、TTS、主题、年度报告、日志工程化、本地多格式整合上有超越潜力；在书源生态、订阅/RSS、Web 服务、源调试、边界成熟度上仍明显落后 Legado。

如果目标是“普通用户稳定看小说和本地漫画”，MoRealm 已经接近可用产品；如果目标是“Legado 生态级替代”，下一步必须集中补书源生态和阅读器边界，而不是继续堆散功能。
