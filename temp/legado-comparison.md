## 1. TTS 通知栏（P0 — 当前最紧急）

### 现状对比

| 维度 | Legado | MoRealm |
|------|--------|---------|
| 通知框架 | `MediaSessionCompat` (legacy) + `NotificationCompat.Builder` 手动构建 | Media3 `MediaSessionService` 自动生成 |
| 标题 | `阅读·朗读: 书名`（暂停→"暂停"，定时→"定时 X 分钟"） | 仅显示书名，无状态文案 |
| 内容文本 | 章节标题 | 章节标题 |
| 封面 | Glide 加载 Bitmap → `setLargeIcon` | 网络 URL 直接下载，无缓存 |
| 副标题 | `setSubText("朗读")` | 无 |
| 按钮数量 | 5 个：上一章 / 播放暂停 / 下一章 / 停止 / 定时 | 依赖 Media3 默认（播放暂停 + 前进后退） |
| 紧凑视图 | `MediaStyle.setShowActionsInCompactView(0,1,2)` 显示前 3 个 | Media3 默认行为 |
| MediaMetadata | ART=封面, TITLE=章节, ARTIST=书名, ALBUM=作者 | `SimpleBasePlayer` 提供基础 metadata |
| 定时显示 | 通知标题动态切换"定时 X 分钟" | 无 |

### 差距分析

- **封面缓存**：Legado 通过 Glide 缓存封面 Bitmap，MoRealm 每次从网络下载，可能导致通知栏封面加载慢或失败。
- **按钮完整性**：缺少"停止"和"定时"独立按钮，用户无法从通知栏直接控制定时。
- **状态文案**：暂停/定时状态未反映到通知标题，用户无法一眼判断当前朗读状态。

### 建议方案

1. 自定义 `MediaNotificationProvider`，手动构建 5 按钮布局。
2. 封面加载改用 Coil（项目已有依赖）缓存 Bitmap，传入 `MediaMetadata.artworkData`。
3. 通知标题根据播放状态动态拼接（朗读中 / 暂停 / 定时 X 分钟）。
4. 补全 MediaMetadata 的 ARTIST、ALBUM 字段。

## 2. 书源引擎

### 现状对比

| 功能模块 | Legado | MoRealm | 状态 |
|----------|--------|---------|------|
| 书源规则解析 | AnalyzeRule 全套 | AnalyzeRule 全套 | 持平 |
| RSS 阅读器 | RssSource / RssArticle / RssStar | 无 | **缺失** |
| 书源订阅 | RuleSub，从 URL 自动拉取更新 | 无 | **缺失** |
| 词典规则 | DictRule + DictDialog | 无 | **缺失** |
| 键盘辅助 | KeyboardAssist | 无 | **缺失** |
| 书评/评论 | ReviewRule | 无 | **缺失** |

### 差距分析

- **RSS**：Legado 将 RSS 作为独立内容源，支持订阅新闻/博客，MoRealm 完全没有此功能线。
- **书源订阅**：Legado 用户可通过 URL 一键导入并自动更新书源，MoRealm 仅支持手动 JSON 导入。
- **词典**：Legado 支持在阅读中长按查词，MoRealm 无此交互。

### 影响评估

RSS 和书源订阅属于"锦上添花"功能，核心阅读体验不受影响，但对重度用户吸引力较大。词典规则使用频率较低，优先级可后置。

## 3. 阅读器

### 现状对比

| 维度 | Legado | MoRealm | 判定 |
|------|--------|---------|------|
| 翻页动画 | 仿真/覆盖/滑动/滚动/无 | 仿真/覆盖/滑动/滚动/无 | 持平 |
| 字体/行距/段距/边距 | 全套自定义 | 全套自定义 | 持平 |
| 样式预设 | 5 套 | 5 套 | 持平 |
| 日夜背景 | 支持 | 支持（含背景图） | 持平 |
| 信息栏 | 6 槽位 | 6 槽位 | 持平 |
| Canvas 录制缓存 | 无 | 有 | **MoRealm 优** |
| 自定义 CSS | 无 | 有 | **MoRealm 优** |
| Alpha 羽化（滚动模式） | 无 | 有 | **MoRealm 优** |
| Quote 分享卡片 | 无 | 有 | **MoRealm 优** |
| 提示色/分隔线颜色自定义 | 有 | 无 | **Legado 优** |
| 跨预设共享布局 | 有 | 无 | **Legado 优** |

### 小结

阅读器是 MoRealm 的强项。Canvas 录制缓存带来更流畅的翻页体验，Alpha 羽化和 Quote 分享卡片是差异化亮点。缺失的提示色自定义和跨预设布局属于细节打磨，非核心短板。

## 8. 替换规则

基本持平。MoRealm 有正则超时保护和灾难性回溯检测，属于安全性优势。

---

## 9. Web 服务 / 远程访问

**Legado 有而 MoRealm 完全缺失**：内置 HTTP 服务器、Web 阅读器、远程 API（书籍/书源/替换规则管理）。这是 Legado 的重要差异化功能，但实现成本较高。

---

## 10. 导入导出

基本持平。MoRealm 优势：文件夹批量导入、EPUB 元数据写入。Legado 优势：二维码导入、直链分享上传。

---

## 11. UI/UX

**MoRealm 全面领先**：Jetpack Compose + Material 3、响应式布局、药丸导航栏、滑动切换 Tab、动态主题色、完整主题编辑器。

---

## 12. 总结与优先级路线图

### MoRealm 优势领域
- 现代 UI (Compose + M3)
- Canvas 录制缓存 / Alpha 羽化
- Edge TTS 内置 + 专属听书 Tab
- 自动分组 / 文件夹导入 / 智能解析
- 正则安全保护

### 需要补齐的差距（按优先级）

| 优先级 | 功能 | 预估工作量 | 用户影响 |
|--------|------|-----------|---------|
| **P0** | TTS 通知栏升级 | 1天 | ✅ 已完成 |
| **P1** | 自动检查新章节 | 2-3天 | 高 |
| **P2** | 搜索历史持久化 | 0.5天 | 中 |
| **P3** | 搜索范围过滤 | 1天 | 中 |
| **P4** | 备份加密 + 选择性恢复 | 2天 | 中 |
| **P5** | Web 服务/远程访问 | 5-7天 | 中（重度用户） |
| **P6** | RSS 阅读器 | 3-5天 | 低 |
| **P7** | 词典规则 | 2天 | 低 |
| **P8** | 书源订阅 | 1天 | 低 |
